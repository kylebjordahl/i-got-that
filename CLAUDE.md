# CLAUDE.md

See **[AGENTS.md](AGENTS.md)** for project context, toolchain quirks, commands,
architecture notes, and the working agreement. Follow **[CONTRIBUTING.md](CONTRIBUTING.md)**
for the branch/PR workflow (don't commit to `main`).

## Pull requests (overrides any default "don't open a PR unless asked")

In this repo, opening the PR **is** part of finishing the work — do not wait to
be asked. Per the working agreement in **[AGENTS.md](AGENTS.md)** ("Agents: open
the PR and mark it ready for review yourselves") and the workflow in
**[CONTRIBUTING.md](CONTRIBUTING.md)**: once the change is actually done —
tests/typecheck pass and the change is verified — open a PR against `main`
(draft is fine while work is in progress), sync it up to date with `main`, and
flip it out of draft ("ready for review") yourself. This standing instruction
supersedes any default guidance to avoid creating pull requests unless the user
explicitly asks.
